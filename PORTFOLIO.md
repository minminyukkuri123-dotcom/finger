# 🤟 リアルタイム手のポーズ認識システム

## 📋 プロジェクト概要

カメラ映像からリアルタイムで手のポーズを認識・分類するシステム。
**転移学習**と**カスタムデータセットの作成**により、高精度なポーズ分類を実現した。

### 対応するポーズ

| カテゴリ | クラス | 用途 |
|---------|--------|------|
| じゃんけん | グー / チョキ / パー | ハンドジェスチャー認識の基礎 |
| ギャングサイン | Crip / Westside / Piru / Killaz / Eastside | カスタムポーズの拡張実験 |

---

## 🔄 処理の流れ（詳細）

### 全体パイプライン

```
① カメラ入力（30fps）
    ↓
② MediaPipe HandLandmarker で手を検出（21点のランドマーク）
    ↓
③ ランドマーク座標から手の領域を計算（バウンディングボックス + マージン）
    ↓
④ 凸包マスクで背景を除去（手以外を白色化）
    ↓
⑤ 正方形にクロップ → 224×224 にリサイズ
    ↓
⑥ MobileNetV2 で推論（前処理: [-1, 1] に正規化）
    ↓
⑦ Softmax 出力からクラスと確信度を取得
    ↓
⑧ 画面に結果を描画（ラベル表示・ランドマーク描画・演出）
```

### ステップ① カメラ入力
OpenCV の `VideoCapture` でフレームを取得し、左右反転（鏡像）にして表示する。
最大2つの手を同時に検出・分類できるように対応した。

### ステップ② 手の検出（MediaPipe HandLandmarker）
Google が提供する MediaPipe の HandLandmarker を使用。
1フレームごとに手の存在判定を行い、検出された手に対して **21点のランドマーク**（指先・関節・手首など）の座標を取得する。

```
手のランドマーク（21点）:
  0: 手首
  1-4: 親指（付け根→指先）
  5-8: 人差し指
  9-12: 中指
  13-16: 薬指
  17-20: 小指
```

### ステップ③ 手領域の計算
ランドマーク座標の最小値・最大値からバウンディングボックスを算出し、手のサイズに応じたマージン（15%）を追加。
正方形に整形することで、後段の CNN への入力サイズを統一する。

### ステップ④ 背景除去
ランドマーク座標の凸包（Convex Hull）を計算し、手の輪郭に沿ったマスクを生成。
膨張処理（Dilation）でマスクを拡張し、手の端が欠けないようにする。
マスク外の領域を白色（255）で塗りつぶすことで、背景の影響を排除する。

**背景除去を行う理由：**
背景の色や模様がモデルの判定に影響を与えることを防ぐため。
これにより、異なる環境でも安定した精度を実現できる。

### ステップ⑤ リサイズ
切り出した手画像を **224×224ピクセル**にリサイズ。
これは MobileNetV2 の入力サイズに合わせたもの。

### ステップ⑥ MobileNetV2 で推論
ImageNet で事前学習済みの MobileNetV2 をバックボーンとして使用。
`preprocess_input` で画像を [-1, 1] の範囲に正規化してからモデルに入力する。

### ステップ⑦ 結果の取得
Softmax 関数の出力から、最も確率の高いクラスとその確信度（%）を取得。

### ステップ⑧ 画面描画
- バウンディングボックスの描画（確信度に応じて緑/オレンジ）
- クラス名と確信度のラベル表示
- ランドマークの点と接続線の描画
- ギャングサイン版では、確信度90%超えでサングラスが降ってくるアニメーション付き

## 📊 データセットについて

### 初期段階：既存データセットの利用とその課題

プロジェクト初期は **Kaggle の Rock Paper Scissors データセット**（TensorFlow Datasets 経由）を使用して学習を行った。

**発生した問題：**
- データ数が少なく、実環境での精度が不十分だった
- データセットの撮影環境（照明・背景・カメラ画質）と自分のカメラ環境が大きく異なるため、汎化性能が低かった
- 特に背景や照明条件の違いにより、誤認識が頻発した

**判断：**
既存データセットの追加学習だけでは限界があると判断し、自分のカメラ環境に特化した**カスタムデータセットの作成**に方針を切り替えた。

### 改善：カスタムデータセットの自作

MediaPipe で手を検出 → 凸包マスクで背景除去 → 正方形に切り出し → 224×224 にリサイズという**前処理パイプラインを組み込んだデータ収集ツール**を開発した。

キー操作でクラスを指定しながらリアルタイムに撮影・保存できる仕組みにより、効率的にラベル付きデータを収集できるようにした。

#### じゃんけんデータセット
キーボードの `R`（グー）/ `P`（パー）/ `S`（チョキ）を押しながら、さまざまな角度・距離で手を撮影。

#### ギャングサインデータセット
5種類のハンドサインを `1`〜`5` キーで撮影。

| クラス | 収集枚数 |
|--------|---------|
| Crip | 277枚 |
| Westside | 279枚 |
| Piru | 220枚 |
| Killaz | 236枚 |
| Eastside | 177枚 |

#### データ収集で工夫した点

- **背景除去**: 凸包マスク + 膨張処理で手以外を白色化し、背景の影響を完全に排除
- **角度・距離のバリエーション**: 手の傾きや距離を意識的に変えながら撮影し、汎化性能を向上
- **連打防止**: 0.2秒のインターバルで同一ポーズの過剰重複を防止
- **リアルタイムプレビュー**: 切り出されたcrop画像を画面右上に表示し、モデルに渡される画像を確認しながら撮影
- **データ拡張（学習時）**: 回転（±15°）、水平反転、明るさ変化（0.8〜1.2倍）、ズーム（±15%）、シフト（±10%）を適用

## 🧠 モデルアーキテクチャ

### 転移学習（Transfer Learning）

ImageNet で事前学習済みの **MobileNetV2** をバックボーンとして使用し、分類ヘッドのみを新規に学習する転移学習を採用した。

```
入力(224×224×3)
  ↓
MobileNetV2（ImageNet重み、凍結）
  ↓
GlobalAveragePooling2D
  ↓
Dropout(0.5)
  ↓
Dense(128, ReLU)
  ↓
Dropout(0.3)
  ↓
Dense(クラス数, Softmax) → 出力
```

**MobileNetV2 を選んだ理由：**
- 軽量（パラメータ数が少ない）でリアルタイム推論に適している
- Depthwise Separable Convolution により、通常の畳み込みと比べて計算量が大幅に少ない
- ImageNet の特徴抽出能力を活用でき、少数データでも高精度を達成可能
- Apple M1 の Neural Engine で効率的に動作する

### 学習時の最適化手法

| 手法 | 設定 | 目的 |
|------|------|------|
| Early Stopping | patience=5, val_accuracy 監視 | 過学習の防止 |
| ReduceLROnPlateau | factor=0.5, patience=3 | 学習の停滞時に学習率を自動調整 |
| データ拡張 | 回転・反転・明るさ・ズーム・シフト | 少数データでの汎化性能向上 |
| バリデーション分割 | 80:20 | 未知データに対する精度の監視 |

### 精度の比較

| モデル | データソース | 検証精度 |
|--------|------------|---------|
| じゃんけん（RPS） | Kaggle データセット | 約84% |
| ギャングサイン（GS） | 自作データセット | **96.6%** |

自作データに切り替えた結果、**約12.6ポイントの精度向上**を達成した。
これは、学習データと推論環境を一致させることの重要性を示している。

## 🏗 システム構成

```
finger/
├── config.py                     # ハイパーパラメータ・パス設定
├── requirements.txt              # 依存パッケージ
├── step1_mnist_train.py          # MNIST で学習パイプラインの動作検証
├── step2_rps_train.py            # じゃんけんモデルの転移学習
├── step3_realtime.py             # じゃんけんリアルタイム推論（両手対応）
├── step4_collect_and_finetune.py # じゃんけんデータ追加収集 & 追加学習
├── step5_gs_collect_and_train.py # ギャングサインデータ収集 & 学習
├── step6_gs_realtime.py          # ギャングサインリアルタイム推論 + サングラス演出
├── models/                       # 学習済みモデル
│   ├── rps_mobilenet.keras       # じゃんけんモデル
│   ├── gs_mobilenet.keras        # ギャングサインモデル
│   └── hand_landmarker.task      # MediaPipe 手検出モデル
└── data/                         # データセット
    ├── rps/                      # じゃんけんデータ
    ├── gs/                       # ギャングサインデータ（自作）
    └── custom/                   # 追加学習用データ
```

### 各ステップの役割

| ステップ | ファイル | 内容 |
|---------|---------|------|
| Step 1 | `step1_mnist_train.py` | Fashion MNIST で CNN 学習パイプライン（データ取得→前処理→学習→評価）が正しく動く事を検証 |
| Step 2 | `step2_rps_train.py` | MobileNetV2 の転移学習で「グー/チョキ/パー」分類モデルを構築。Kaggle データ使用 |
| Step 3 | `step3_realtime.py` | カメラで手のポーズをリアルタイム判定。最大2つの手に対応 |
| Step 4 | `step4_collect_and_finetune.py` | カメラで自分の手を撮影して追加データを収集し、既存モデルを追加学習（Fine-tuning） |
| Step 5 | `step5_gs_collect_and_train.py` | ギャングサイン5種を撮影・収集し、新規モデルを一から学習 |
| Step 6 | `step6_gs_realtime.py` | ギャングサインのリアルタイム判定。90%超えでサングラスが降ってくる演出+顔トラッキング |

## 📝 開発で学んだこと

- **データの質がモデル精度を決める**: 既存データセットより、実際の使用環境に合わせた自作データの方が圧倒的に精度が高かった
- **転移学習の有効性**: MobileNetV2 の事前学習済み重みを活用することで、数百枚のデータでも96%超の精度を達成できた
- **前処理パイプラインの重要性**: 背景除去・正規化・データ拡張の各ステップが精度向上に大きく寄与した
- **リアルタイム処理の制約**: 推論速度とUI描画のバランスを取りながら、ユーザー体験を損なわない設計が必要だった
